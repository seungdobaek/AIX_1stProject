자료와 정리해 둔 프로젝트 기획을 기준으로 보면, 방향은 매우 잘 잡혀 있고 “현업형 STLF 프로젝트” 구조를 이미 갖추고 있습니다.
아래에서는 **이미 적어둔 전처리 외에 “추가하면 확실히 점수가 올라가는 전처리/검증 포인트”**만 선별해서 제안하겠습니다.
(불필요하게 과한 딥러닝 전처리는 의도적으로 제외했습니다.)

1. 시간 축 정합성 전처리 (반드시 추가 권장)
1-1. 시간 누락·중복 검증

전력 데이터 + 기상 데이터는 시간 단위 병합이므로 아래 검증은 필수입니다.

법정동별로

YYYY-MM-DD HH 기준 연속 24시간 × 일수가 맞는지

특정 시간대 누락 / 중복 여부 확인

권장 처리

누락 시간:

전력량 → 직전 시간 보간(forward fill) 또는 같은 요일·시간 평균

기상 → 이미 정한 대로 강수·적설은 0, 기온·습도는 선형 보간

중복 시간:

평균값으로 집계 (sum ❌)

👉 이 작업은 모델 성능보다 “데이터 신뢰성” 차원에서 매우 중요합니다.

2. 전력량 데이터 전처리 보강
2-1. 이상치(outlier) 완화 (제거 ❌, 완화 ⭕)

전력량은 실제 수요이므로 제거하면 안 됩니다.

추천 방식

법정동별·시간대별로

상위 99.5% 이상 값 → winsorizing

또는 log1p(전력량) 파생 변수 생성 (원본은 유지)

모델 입력은 log_power,
결과 해석/시각화는 원본 전력량

2-2. “0 또는 급락 구간” 플래그 변수

특정 시간에 전력량이 비정상적으로 낮은 경우가 있을 가능성이 큼.

추가 변수

is_low_power = 1 if 전력량 < (법정동별 하위 1~2%) else 0


👉 모델이 “이상 상태”를 학습하지 않도록 돕는 보조 신호 변수

3. 기상 데이터 전처리 고도화 (중요)
3-1. 체감 관련 파생 변수 (강력 추천)

이미 불쾌지수 언급했는데, 이건 꼭 넣는 게 좋습니다.

(1) 불쾌지수 (여름)
DI = 0.81*T + 0.01*RH*(0.99*T - 14.3) + 46.3

(2) 체감온도 / 냉난방 부하 변수

간단한 방식으로도 충분합니다.

cooling_degree = max(기온 - 24, 0)
heating_degree = max(18 - 기온, 0)


👉 전력 예측에서 기온 원본보다 설명력이 훨씬 큼

3-2. 날씨 “변화량” 변수

사람과 전력은 절대값보다 변화에 반응합니다.

추가 추천 변수:

temp_diff_1h = 현재기온 - 1시간 전 기온

temp_diff_24h = 현재기온 - 24시간 전 기온

rain_yn = 1 if 강수량 > 0 else 0

4. 법정동 코드 처리 관련 제안
4-1. Label Encoding만 쓰지 말 것

기획에는 Label Encoding / One-Hot이 언급되어 있는데, 보완 제안입니다.

추천 전략 (ML 기준)

기본: Label Encoding

추가: 법정동별 통계 Feature

dong_mean_power
dong_std_power
dong_peak_hour


→ 데이터 누수 방지를 위해

반드시 train 기간(22~24년) 기준으로 계산

👉 이 한 단계가 “지역 패턴 분리 학습”의 핵심 차별화 포인트입니다.

5. Train/Test 분리 시 전처리 주의점 (매우 중요)
5-1. 전처리는 반드시 시간 기준 분리 후 수행

아래는 포트폴리오에서 자주 지적받는 부분입니다.

❌ 전체 데이터에서 스케일링
⭕ Train(22~24) 에서 fit → Test(25) 에 transform

적용 대상:

StandardScaler / MinMaxScaler

법정동 통계 Feature

이상치 기준값

6. “지금은 넣지 말되, 다음 단계용” 전처리

이미 기획서에 적어둔 내용과 잘 맞습니다.

LAG Feature

power_t-1, power_t-24, power_t-168

Rolling Feature

rolling_mean_24h, rolling_std_24h

👉 초기 모델 제외 → 성능 개선 증명용 카드로 아주 좋음

7. 전처리 단계 요약 (정리본)

기존 + 추가 통합 전처리 플로우

시간 포맷 통일 및 누락·중복 검증

전력량 이상치 완화 (winsorize / log)

기상 결측치 처리 (0 / 보간)

시간 파생 변수 (요일·월·시간·공휴일)

기상 파생 변수 (불쾌지수, 냉난방지수, 변화량)

법정동 통계 Feature (train 기준)

Train/Test 분리 후 스케일링
